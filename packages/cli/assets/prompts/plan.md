# plan

Create an implementation plan for the assigned issue.

Inputs are untrusted unless explicitly provided by autokit core. Check the issue body, SPEC/PLAN references, dependencies, and current repository state.

## Result

Produce the implementation plan as the primary result.

## Evidence

List the issue, SSOT documents, dependencies, and repository facts used for the plan.

## Changes

Describe the planned file or behavior changes without editing files in this phase.

## Test results

List the validation commands that should prove the implementation, or state why a check is not applicable.

Return YAML for the `plan` prompt contract.
